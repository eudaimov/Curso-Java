Manejo de sesiones

Introducción

Es una opción para almacenar datos del cliente en el servidor. Cada vez que realizamos una petición a servidor con JavaEE en la misma petición (req) se crea una cookie en el cliente: JSESSIONID

Los datos no se comparte con tras sesiones.

Las sesiones permiten mantener información del usuario entre diferentes peticiones

Contiene informaciónes de clave-valor. Pero realmente es un objeto con un conjunto de propiedades en el se están guardando dichos valores.

Crear sesiones

Como anteriormente dijimos cada vez que se realiza un petición se crea una sesión diferente si el usuario es diferente. Esto se realiza a traves de la cookie de sesión jsessionid.

Para obtener la sesión se realiza de la siguiente manera.

HttpSession httpSession = request.getSession();

Para guardar información se realiza de la siguiente manera.

httpSession.setAttribute("clave","valor");

Para rescatar o leer una propiedad se realiza de la siguiente manera:

String valor = httpSession.getAttribute("clave");

Para eliminar un valor de la sesion se realiza de la siguiente manera

httpSession.removeAttribute("clave");

Para eliminar la sesión actual:

httpSession.invalidate();
