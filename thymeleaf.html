Thymeleaf

¿Qué es?

Thymeleaf es un moderno motor de plantillas Java del lado del servidor para entornos web e independientes, capaz de procesar HTML, XML, JavaScript, CSS e incluso texto sin formato.

El objetivo principal de Thymeleaf es proporcionar una forma elegante y fácil de mantener de crear plantillas. Para lograr esto, se basa en el concepto de plantillas naturales para inyectar su lógica en los archivos de plantilla de una manera que no afecte el uso de la plantilla como prototipo de diseño. Esto mejora la comunicación del diseño y cierra la brecha entre los equipos de diseño y desarrollo.

Thymeleaf también ha sido diseñado desde el principio teniendo en cuenta los estándares web, especialmente HTML5 , lo que le permite crear plantillas completamente validadas si así lo necesita.

Uso

Agregaremos xmlns:th atributo a nuestra <html>etiqueta, algo que no es en absoluto HTML5:

<html xmlns:th="http://www.thymeleaf.org">

Usando th:text y externalizando texto

Externalizar texto consiste en extraer fragmentos de código de plantilla de los archivos de plantilla para que puedan guardarse en archivos separados (normalmente messages.properties) y puedan reemplazarse fácilmente con textos equivalentes escritos en otros idiomas (un proceso llamado internacionalización o simplemente i18n ).

Para cambiar el archivo de mensaje se realizar application.properties y se pone lo siguiente

spring.messages.basename=nombreArchivo;

Los fragmentos de texto externalizados suelen denominarse “mensajes”.

Los mensajes siempre tienen una clave que los identifica, y Thymeleaf te permite especificar que un texto debe corresponder a un mensaje específico con la #{...}sintaxis:

<p th:text="#{home.welcome}">Welcome to our grocery store!</p>

El Controlador: @Controller

Diferencias entre @Controller y @RestController

La principal diferencia entre usar @Controller y @RestController en Thymeleaf reside en el tipo de respuesta que se genera y en cómo se maneja la renderización de vistas.

@Controller:

Se utiliza para crear controladores tradicionales de Spring MVC que devuelven vistas (como HTML, JSP o Thymeleaf) al cliente.

El controlador interactúa con un resolvedor de vistas para mapear el valor de retorno del método a una plantilla de vista específica.

El resolvedor de vistas procesa la plantilla y la combina con los datos del modelo para generar una respuesta HTML completa.

@RestController:

Es una anotación especializada para crear controladores RESTful que devuelven datos directamente al cliente en formato JSON o XML.

Combina las anotaciones @Controller y @ResponseBody, lo que significa que el valor de retorno del método se serializa automáticamente en el formato de respuesta deseado.

No se utiliza un resolvedor de vistas, ya que la respuesta es solo un conjunto de datos.

En resumen:

Para utilizar Thymeleaf lo adecuado es utilizar @Controller el cual podrás utilizar un modelo(Model) con el que comunicarte con la vista

Model

Es un propiedad/clase que va en el método del controlador y nos sirve para comunicarnos con la vista. Alguno de los métodos más utilizados son:

model.addAttribute("saludo", "holaMundo"}

Se puede pasar un texto, como cualquier otra cosa, objeto/s, colecciones, número/s ....

Las principales ventajas de usar el Model en Thymeleaf:

  • Separación de preocupaciones: Promueve la separación entre la lógica del servidor y la presentación de la vista.
  • Reutilización de código: Permite compartir datos entre diferentes partes de la aplicación.
  • Plantillas más limpias y expresivas: Reduce la necesidad de código Java incrustado en las plantillas Thymeleaf.

La plantilla Thymeleaf puede acceder a los datos del Model utilizando expresiones Thymeleaf, como:

${atributoModel}
