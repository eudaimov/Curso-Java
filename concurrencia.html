Concurrencia en Java

La concurrencia es la capacidad de un sistema para ejecutar dos o más tareas al mismo tiempo. En Java, la concurrencia se puede implementar de varias maneras, incluyendo:

  1. Hilos (Threads):
  2. Sincronización
  3. ExecutorService
  4. CompletableFuture

Los usuarios de computadoras dan por sentado que sus sistemas pueden hacer más de una cosa a la vez. Suponen que pueden seguir trabajando en un procesador de textos, mientras otras aplicaciones descargan archivos, gestionan la cola de impresión y transmiten audio. A menudo se espera que incluso una sola aplicación haga más de una cosa a la vez. Por ejemplo, esa aplicación de streaming de audio debe leer simultáneamente el audio digital de la red, descomprimirlo, gestionar la reproducción y actualizar su visualización. Incluso el procesador de textos siempre debe estar listo para responder a los eventos del teclado y el mouse, sin importar cuán ocupado esté reformateando el texto o actualizando la pantalla. El software que puede hacer este tipo de cosas se conoce como software concurrente.

La plataforma Java está diseñada desde cero para admitir programación concurrente, con soporte de concurrencia básico en el lenguaje de programación Java y las bibliotecas de clases Java. Desde la versión 5.0, la plataforma Java también incluye API de simultaneidad de alto nivel. Esta lección presenta el soporte de concurrencia básico de la plataforma y resume algunas de las API de alto nivel en los java.util.concurrentpaquetes.
