Jakarta EE

¿Qué es Jakarta EE

Jakarta EE, anteriormente conocido como Java EE (Java Platform, Enterprise Edition), es una plataforma de desarrollo de software empresarial basada en Java. Fue desarrollada originalmente por Sun Microsystems (posteriormente adquirida por Oracle Corporation) y ahora es mantenida por la comunidad de software libre a través de la Eclipse Foundation.

Ofrece un conjunto de especificaciones y API que facilita la creación de aplicaciones escalables, robustas y seguras

Estas especificaciones abarcan una amplia gama de tecnologías para el desarrollo de aplicaciones empresariales, incluyendo:

  • Servlets: Para el desarrollo de aplicaciones web.
  • JavaServer Pages (JSP): Para la creación de vistas dinámicas en aplicaciones web.
  • Enterprise JavaBeans (EJB): Para la creación de componentes de negocio y lógica empresarial.
  • Java Persistence API (JPA): Para el acceso a bases de datos de manera orientada a objetos.
  • Java Message Service (JMS): Para la comunicación asíncrona entre componentes de la aplicación.
  • Java API for RESTful Web Services (JAX-RS): Para el desarrollo de servicios web RESTful.
  • JavaServer Faces (JSF): Para la creación de interfaces de usuario basadas en componentes.

Jakarta EE pueden ejecutarse en una amplia gama de servidores de aplicaciones compatibles con Java EE, lo que permite a los desarrolladores desplegar sus aplicaciones en diferentes entornos de manera flexible.

Creacion de un proyecto Jakarta con maven

Creamos un proyecto Java con Maven

Configuramos el pom de la siguiente manera

<?xml version="1.0" encoding="UTF-8"?> <project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd"> <modelVersion>4.0.0</modelVersion> <groupId>es.felix</groupId> <artifactId>JakartaEjemplo</artifactId> <version>1.0-SNAPSHOT</version> <name>JakartaEjemplo</name> <packaging>war</packaging> <properties> <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding> <maven.compiler.target>11</maven.compiler.target> <maven.compiler.source>11</maven.compiler.source> <junit.version>5.9.2</junit.version> </properties> <dependencies> <dependency> <groupId>jakarta.platform</groupId> <artifactId>jakarta.jakartaee-api</artifactId> <version>9.1.0</version> <scope>provided</scope> </dependency> <dependency> <groupId>org.hibernate.orm</groupId> <artifactId>hibernate-core</artifactId> <version>6.0.2.Final</version> </dependency> <dependency> <groupId>org.glassfish.jaxb</groupId> <artifactId>jaxb-runtime</artifactId> <version>3.0.2</version> </dependency> <dependency> <groupId>org.junit.jupiter</groupId> <artifactId>junit-jupiter-api</artifactId> <version>${junit.version}</version> <scope>test</scope> </dependency> <dependency> <groupId>org.junit.jupiter</groupId> <artifactId>junit-jupiter-engine</artifactId> <version>${junit.version}</version> <scope>test</scope> </dependency> </dependencies> <build> <plugins> <plugin> <groupId>org.apache.maven.plugins</groupId> <artifactId>maven-war-plugin</artifactId> <version>3.3.2</version> <configuration> <!-- Si falta el archivo web.xml dentro de WEB-INF se salta este error--> <failOnMissingWebXml>false</failOnMissingWebXml> </configuration> </plugin> </plugins> </build> </project>
