Clases

Definición:

Las clases son los modelos o plantillas necesarios para la creación de objetos. Por lo que en resumidas cuentas modela el estado y comportamiento de objetos reales. Es la esencia de la programción orientada a objetos en Java.

Una clase tiene principalmente tres componentes:

  • Atributos, propiedades o campos.
  • Métodos o funciones
  • Constructores

Declaración

Debe corresponder una sóla clase por archivo y debe tener el mismo nombre la clase que el archivo.

La clase debe tener su primera letra en Mayúscula y la palabra tiene que estar en singular(por convención). Corresponde al nombre de un objecto. Si el nombre se compone de dos palabras la segunda palabra debe aparecer junto a la primera sin espacios y con su primera letra en mayúsculas. Es lo que se conoce como CamelCase.

Se construye de la siguiente manera.

  • Modificadores de acceso. Suele ser siempre public.
  • class
  • NombreClase. El nombre de la clase.
    • Optionales:
  • extends NombreClase. La clase de la que extiende. Solamente puede extender de una sóla clase.
  • implements NameClass. Las interfaces que implementa. Si son varias puedes separarlas con coma.
  • { ... }. Llave donde se cololcan todos los componentes con los cuales se construye una clase (atributos, métodos y constructores).

Por ejemplo:

class MyClass extends MySuperClass implements YourInterface { // field, constructor, and method declarations }

Atributos o propiedades de una clase.

Hay varios tipos de variables que podemos encontrar en una clase: Atributos, variables locales y parametros o argumentos de los métodos.

En esta sección nos centraremos en los atributos de una clase.

Se suele declarar de la siguiente manera

  1. Modificador de acceso: private("El más utilizado"), protect, public, default.
  2. Tipo de Objeto: primitivo, clase ...
  3. Nombre de la propiedad: Suele ser siempre en minúscula. No se tienen porque inicializar a no ser que quieras que coja un valor por defecto.

Métodos o funciones

Son las acciones de nuestras clases. Los únicos elementos requeridos de una declaración de método son el tipo de retorno del método, el nombre, un par de paréntesis ()y un cuerpo entre llaves {}.

Constructores

Tiene el mismo nombre que la clase puede tener modificadores de acceso pero carecen de un tipo de devolución explicito pues devuelven el propio la misma clase convertida en objeto, es decir, una instancia. Por lo general, la función de los constructores es aasignar valores iniciales a las variables.

El modificador por defecto es public aunque se podría poner private si queremos evitar que se pueda instanciar y utilizar unicamente métodos estáticos.

Puede existir una sobrecarga de métodos que se diferenciaran unos de otros por los parametros de este.

Todas las clases tienen constructores los defina o no, ya que Java ofrece automáticamente constructores predeterminados o también conocidos constructores por defecto que inicializa todas las varialbes en us valores predetermindos 0, null o false.

Veamos un ejemplo:

public class MyClass{ String nombre; //Constructor: public MyClass(){ this.nombre="Anónima"; } }

En este ultimo ejemplo el constructor asigna a la propiedad nombre el valor por defecto Anónima.

Una vez creado un constructor, el constructor por defecto desaparece y el atributo que anteriormente sería null por el constructor por defecto desaparece.
