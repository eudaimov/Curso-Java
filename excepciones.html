Excepciones

Introducción:

¿Qué es una excepción?

Una excepción es un problema o evento que ocurre durante la ejecución del programa que interrumpe el flujo normal

Cuando ocurre un error dentro de un método, el método crea un objeto y lo entrega al sistema de ejecución. El objeto, llamado objeto de excepción, contiene información sobre el error, incluido su tipo y el estado del programa cuando ocurrió el error. Crear un objeto de excepción y entregarlo al sistema de ejecución se denomina lanzar una excepción.

Después de que un método genera una excepción, el sistema de ejecución intenta encontrar algo para manejarla. El conjunto de posibles "algos" para manejar la excepción es la lista ordenada de métodos que se llamaron para llegar al método donde ocurrió el error. La lista de métodos se conoce como pila de llamadas.

Las principales características del tratamiento de excepciones en Java es:

  • Separa el código que gestiona los errores del código principal.
  • Nos permite manejar el error y continuar con la ejecución del programa
  • Agrupar y diferenciar entre diferentes tipos de errores
  • Propagar errores hacia arriba en la pila de llamadas.

¿Capturar o especificar?

El código válido del lenguaje de programación Java debe cumplir con el requisito de captura o especificación . Esto significa que el código que podría generar ciertas excepciones debe estar encerrado en cualquiera de los siguientes:

  • Una try declaración que detecta la excepción. Debe try proporcionar un controlador para la excepción, como se describe en Captura y manejo de excepciones. Un método que especifica que puede generar la excepción. El método debe proporcionar una throwscláusula que enumere la excepción, como se describe en Especificación de las excepciones generadas por un método.
  • Un método que especifica que puede generar la excepción. El método debe proporcionar una cláusula throws que enumere la excepción, como se describe en Especificación de las excepciones generadas por un método.

Tipos de Errores

Existen tres tipos de errores:

  1. Marcada o comprobada
  2. Errores ajenos a la aplicación.
  3. Errores en tiempo de ejecución (RunTimeException) o no comprobadas

Dentro de las inesperadas se encuentran las que son ajenas a java y otras que se encuentra dentro de Java que son conocidas como RunTimeException o excepciones ocurridas en Tiempo de Ejecución.También se le conoce como excepciones no comprobadas.
